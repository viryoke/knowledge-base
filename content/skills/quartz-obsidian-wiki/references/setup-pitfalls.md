# Quartz 5 Setup Pitfalls

Real issues encountered setting up Quartz 5 with an Obsidian vault on macOS with TUN-mode proxy.

## Git Clone Timeout

**Problem**: `git clone https://github.com/jackyzha0/quartz.git` hangs indefinitely, even with TUN-mode proxy (Clash Verge).

**Root cause**: TUN mode handles HTTPS but git's fetch-pack protocol can be disrupted.

**Fix**: Download zip instead:
```bash
curl -L -o quartz.zip https://github.com/jackyzha0/quartz/archive/refs/heads/v5.zip
unzip quartz.zip && mv quartz-5 quartz
```

Do NOT set `git config --global http.proxy` when using TUN mode — it conflicts.

## Missing .quartz/plugins/index.ts

**Problem**: `npx quartz build` fails with `Could not resolve "../../.quartz/plugins"`.

**Root cause**: The `.quartz/plugins/index.ts` barrel file is normally generated by `npx quartz plugin install`, but that command requires network access to clone plugin repos (which may timeout).

**Fix**: Create minimal barrel file manually:
```typescript
// .quartz/plugins/index.ts
export { CustomOgImagesEmitterName } from "./og-image"
```

The plugins directory already exists from the zip download — only the barrel file is missing.

## SCSS @import Ordering Conflict

**Problem**: Build fails with `@import rules must precede all rules aside from @charset and @layer statements` when adding font CDN URLs via `@import url()` in `custom.scss`.

**Root cause**: Quartz concatenates multiple SCSS files. The `@use "./variables.scss"` in custom.scss must come before any other rules, but `@import url()` for fonts must come before `@use` in standard CSS — creating an impossible ordering.

**Fix**: Do NOT use `@import url()` in custom.scss. Instead:
1. Set `theme.fontOrigin: googleFonts` in quartz.config.yaml
2. Add Google Fonts via the config's typography section
3. For non-Google fonts (e.g., LXGW WenKai), inject the CDN link via layout components or a custom Head override

**What NOT to try**: Reordering @use and @import — both orderings fail with different errors.

## satori Dependency Missing

**Problem**: og-image plugin fails with `Cannot find module 'satori'`.

**Fix**: `npm install satori` in the quartz project root.

## CSS Variable Names

**Problem**: Custom fonts don't apply even though CSS variables are set.

**Root cause**: Quartz uses `--bodyFont`, `--headerFont`, `--codeFont` — NOT `--font-body`, `--font-header`, etc.

**Fix**: Use the correct variable names in custom.scss:
```scss
:root {
  --bodyFont: "LXGW WenKai", "Inter", sans-serif;
  --headerFont: "LXGW WenKai", "Inter", sans-serif;
  --codeFont: "JetBrains Mono", monospace;
}
```

Verify with browser DevTools: `getComputedStyle(document.body).fontFamily`

## Toolsets Require /reset

**Problem**: After `hermes tools enable image_gen`, the tool doesn't appear in the current session.

**Fix**: Tool changes require `/reset` (new session) to load. They do NOT apply mid-conversation to preserve prompt caching.
