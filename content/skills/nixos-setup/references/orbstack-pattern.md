# OrbStack Container Pattern

OrbStack's NixOS implementation is a **container (LXC)**, not a traditional VM. This affects how you apply configurations.

## Key Characteristics

1. OrbStack manages `/etc/nixos/configuration.nix`
2. System reboots can revert to original profiles (system-1-link, system-2-link)
3. Custom nixos-rebuild profiles (system-3-link, etc.) may not persist
4. Solution: Import your config as a **module** into OrbStack's configuration

## Setup Workflow

### Step 1: Clone your config repo
```bash
git clone https://github.com/user/nix-config.git ~/nix-config
```

Or if git is not available:
```bash
nix-shell -p git --run 'git clone https://github.com/user/nix-config.git ~/nix-config'
```

### Step 2: Copy custom.nix to /etc/nixos/
```bash
sudo cp ~/nix-config/hosts/nixos/custom.nix /etc/nixos/
```

### Step 3: Add import to configuration.nix
```bash
sudo sed -i 's|./orbstack.nix|./orbstack.nix\n      ./custom.nix|' /etc/nixos/configuration.nix
```

Verify the import was added:
```bash
head -20 /etc/nixos/configuration.nix | grep -A5 "imports ="
```

Expected output:
```nix
imports = [
  # Include the default lxd configuration.
  "${modulesPath}/virtualisation/lxc-container.nix"
  # Include the container-specific autogenerated configuration.
  ./incus.nix
  # Include the OrbStack-specific configuration.
  ./orbstack.nix
  ./custom.nix
];
```

### Step 4: Apply configuration
```bash
sudo nixos-rebuild switch
```

## custom.nix Structure

The `custom.nix` file should be a standalone NixOS module:

```nix
# Custom module imported by OrbStack's configuration.nix
# This file adds our packages and settings to the OrbStack container
{ config, pkgs, lib, ... }:
{
  # System packages (CLI tools)
  environment.systemPackages = with pkgs; [
    # Core CLI tools
    ripgrep
    fd
    eza
    zoxide
    jq
    tree
    htop
    btop
    tldr
    zellij
    git
    curl
    wget
    unzip
    p7zip
    
    # Editors
    neovim
    vim
    
    # Development languages & tools
    python312
    uv
    nodejs_22
    bun
    
    # AI coding agents
    claude-code
  ];
  
  # Fonts
  fonts = {
    fontDir.enable = true;
    packages = with pkgs; [
      nerd-fonts.jetbrains-mono
      noto-fonts-cjk-sans
      noto-fonts-cjk-serif
      noto-fonts-color-emoji
      source-han-sans
      source-han-serif
    ];
    fontconfig.defaultFonts = {
      serif = [ "Source Han Serif SC" "Noto Serif" ];
      sansSerif = [ "Source Han Sans SC" "Noto Sans" ];
      monospace = [ "JetBrainsMono Nerd Font" "Noto Sans Mono CJK SC" ];
      emoji = [ "Noto Color Emoji" ];
    };
  };
  
  # Shell
  programs.zsh.enable = true;
  
  # Docker
  virtualisation.docker.enable = true;
  
  # ZRAM Swap
  zramSwap = {
    enable = true;
    algorithm = "zstd";
    memoryPercent = 25;
  };
  
  # Nix settings
  nix.settings = {
    experimental-features = [ "nix-command" "flakes" ];
    auto-optimise-store = true;
    substituters = [
      "https://mirrors.ustc.edu.cn/nix-channels/store"
      "https://mirrors.tuna.tsinghua.edu.cn/nix-channels/store"
      "https://cache.nixos.org"
    ];
  };
  nix.gc = {
    automatic = true;
    dates = "weekly";
    options = "--delete-older-than 7d";
  };
  
  # Allow unfree packages
  nixpkgs.config.allowUnfree = true;
}
```

## Verification

After applying, verify tools are available:
```bash
which nvim zellij rg eza
nvim --version
docker --version
fc-list : family | grep -i 'jetbrains\|source han\|noto cjk'
```

## Common Issues

### Issue: Tools not available after rebuild
**Cause:** OrbStack reverted to original profile on reboot
**Fix:** Re-run the setup workflow (Step 3 + Step 4)

### Issue: Permission denied when copying custom.nix
**Cause:** Not using sudo
**Fix:** Use `sudo cp`

### Issue: syntax error in configuration.nix
**Cause:** Duplicate imports or malformed imports list
**Fix:** Check `/etc/nixos/configuration.nix` manually, ensure `./custom.nix` appears only once in imports list

### Issue: GitHub API rate limit when cloning
**Cause:** OrbStack VM doesn't have git by default, using nix-shell
**Fix:** Configure GitHub token:
```bash
nix-shell -p git --run 'export GITHUB_TOKEN=*** && git clone https://github.com/user/nix-config.git ~/nix-config'
```

## Differences from Traditional NixOS VMs

| Aspect | OrbStack Container | Traditional VM |
|--------|-------------------|----------------|
| Bootloader | Managed by OrbStack | User configures |
| Kernel | Managed by OrbStack | User configures |
| /etc/nixos/ | OrbStack may overwrite | User controls |
| Profile persistence | May revert on reboot | Persists |
| Hardware detection | Automatic | Automatic |
| Custom config | Import as module | Direct edit |

## End-to-End Validation

To validate the OrbStack pattern works:

1. **Clean slate**: Remove custom config
```bash
sudo sed -i '/custom.nix/d' /etc/nixos/configuration.nix
sudo rm -f /etc/nixos/custom.nix
sudo nixos-rebuild switch
```

2. **Verify clean**: Tools should be gone
```bash
which nvim  # Should return nothing
```

3. **Apply from scratch**: Follow Steps 1-4 above

4. **Verify applied**: Tools should be available
```bash
which nvim zellij rg eza  # Should return paths
```

This workflow ensures the OrbStack container pattern is reproducible and reliable.
