以 Transformer 架构为例，对比 6 种图表语言的表现力

// 渲染原理

Kroki 是一个统一的图表渲染服务。它将各种 Diagram as Code 语言的源代码 发送给对应的渲染引擎（Mermaid → Puppeteer, PlantUML → Java, D2 → Go CLI 等）， 返回 SVG 或 PNG 格式的图形。本地无需安装任何渲染依赖，只需一个 HTTP 请求。

Diagram Code
→
Kroki API
→
Mermaid / PlantUML / D2 ...
→
SVG / PNG

使用方式：将图表代码进行 deflate + base64url 编码后拼接到 URL 中， 如 https://kroki.io/mermaid/svg/{encoded}。 也可自建 Docker 容器（docker run -p 8080:8080 yuzutech/kroki）。

// 支持的 20+ 种图表语言

Mermaid PlantUML D2 GraphViz BlockDiag SeqDiag ActDiag NwDiag PacketDiag RackDiag C4-PlantUML Excalidraw ErD Nomnoml Pikchr Structurizr SVGBob UMlet WireViz Bytefield

所有图表语言通过统一的 API 接口渲染：https://kroki.io/{language}/{format}/{encoded}。 其中 {language} 为图表语言名，{format} 为输出格式（svg/png/pdf）。
