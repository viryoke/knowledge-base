1. 统一AI工具的输出风格

问题：不同AI工具（Hermes、OpenCode、Claude Code）输出风格不一致，需要反复调整。

解决方案：全局 AGENTS.md

构建一份全局的 AGENTS.md，定义AI的角色、行为规范和个人偏好，让各个工具统一引用。

1
定义核心规则 在 AGENTS.md 中声明AI的角色、工作流、禁止事项、输出格式
2
配置引用路径 各工具通过软链接或直接引用，读取同一份 AGENTS.md
3
自动注入上下文 每次会话启动时，AGENTS.md 自动加载到AI的系统提示中

实际配置示例

Hermes Agent

~/.agents/AGENTS.md

通过 SOUL.md 引用，定义学者型助手角色

OpenCode

~/.agents/AGENTS.md

编码规范、注释语言、文件操作约束

Claude Code

~/.claude/CLAUDE.md

代码审查标准、测试要求、输出风格

效果：所有工具输出一致的风格——中文注释、保留注释代码、文本优先、避免空泛建议。

2. 让AI工具更懂你

问题：每次对话都要重新解释背景、偏好、目标，效率低下。

解决方案：个人画像 + 持续记忆

1
Hermes Agent 记录记忆 在交互过程中，Agent 自动识别并记录你的偏好、习惯、个人信息
2
提供个人文档 主动提供简历、目标规划、技术栈等文档，让AI学习并产生记忆
3
生成个人画像 基于积累的 memory，让AI生成结构化的个人画像文件
4
配置到 AGENTS.md 在 AGENTS.md 中声明何时引用画像、何时更新画像

个人画像结构示例

profile/
├── identity.md      # 身份、性格、价值观（极少变）
├── preferences.md   # 交互偏好、输出风格（极少变）
├── tech-stack.md    # 技术栈、学习阶段（中等频率）
├── goals.md         # 当前目标、项目进度（高频变）
└── interests.md     # 兴趣爱好、关注话题（低频变）

AGENTS.md 中的引用规则

## 个人画像引用规则

- 当任务涉及个人目标、职业规划、学习计划时，引用 profile/goals.md
- 当任务涉及技术选型、工具推荐时，引用 profile/tech-stack.md
- 当发现新的偏好信号时（用户纠正风格、表达喜好），更新对应画像文件
- 画像更新后，通知用户审核确认

效果：AI主动推荐符合你当前目标的资源，输出风格自动匹配你的偏好，无需重复解释背景。

3. 知识积累：Karpathy LLM Wiki

问题：AI工具产生的洞察、方案、设计稿散落在各处，无法形成复利。

核心理念（来自 Andrej Karpathy）

为什么不用 RAG？

RAG 每次查询都从零发现知识，重复劳动，交叉引用缺失。

Wiki 的优势：编译一次，持续更新。交叉引用已建好，矛盾已标记，知识网络随时间增值。

三层架构

Layer 3 — Schema（SCHEMA.md）
  ↓ 约束
Layer 2 — Wiki（navigation/, concepts/, entities/, comparisons/）
  ↑ 只读
Layer 1 — Raw Sources（raw/，不可变）

在 AGENTS.md 中声明引用规则

## 本地知识库引用规则

**何时引用知识库：**
- 当任务涉及已收录的课程、概念、人物时，先查询 index.md 和相关页面
- 当生成学习材料、技术文档时，参考已有页面的格式和风格
- 当回答问题时，优先从知识库中检索相关内容

**何时更新知识库：**
- 从新的学习材料（课程、文章、论文）中提取知识时
- 发现新的概念、人物、组织值得存档时
- 产生有价值的问答、对比分析、设计方案时
- 用户明确要求"记录这个"时

**更新流程：**
1. 创建/更新 Wiki 页面
2. 更新 index.md（新增条目 + 更新总数）
3. 追加 log.md（记录操作）
4. 更新相关页面的交叉引用

自动化任务（Hermes 定时任务）

  • AI双日简报：每两天推送AI领域最新动态
  • 知识库 Lint 检查：检测孤儿页面、断链、低置信度页面
  • GitHub 同步：自动推送变更到私有仓库，跨设备共享

已知问题：每次引用知识库会消耗额外 token（上下文浪费），目前尚未找到最优解。

4. 必装的5个Skill

Skill 是 Hermes Agent 的可复用操作流程，将经验转化为可执行资产。

find-skills

发现

搜索是否有现成的 skill 可以解决当前问题，避免重复造轮子。

触发场景：面对新任务时，先调用此 skill 查找可用资源。

skill-creator

创建

高质量创建自定义 skill，遵循 Agent Skills 开放标准。

触发场景：识别到提炼信号时（用户纠正、复杂任务成功、非平凡方案）。

SuperPowers

质量保障

头脑风暴 → Spec → Plan → Verify 四步走，确保AI生成质量。

触发场景：复杂需求、架构设计、需要多轮验证的任务。

frontend-design

设计

生成 web 页面时，减少"AI味"，提升设计质感。

触发场景：创建 landing page、仪表盘、单页应用。

ui-ux-pro-max

素材库

高质量 UI 框架设计素材库，搭配 frontend-design 使用。

触发场景：需要设计系统、组件库、全局架构时。

协作模式：
ui-ux-pro-max 设计全局架构和组件规范 → frontend-design 基于规范生成高质量单页面 → SuperPowers 确保交付质量。

5. 知识库的跨端共享与可视化

问题：知识库只在本机可用，无法在手机、平板或其他设备上随时查阅。

解决方案：三层共享架构

1
GitHub 私有仓库 将 Obsidian Vault 同步到 GitHub 私仓，多设备通过 git pull 获取最新内容。
git remote add origin https://github.com/viryoke/obsidian-vault
2
Quartz 发布到 GitHub Pages 使用 Quartz 5 将可公开的内容构建为静态站点，部署到 GitHub Pages。
配置排除规则：不发布 raw/、DailyNotes、profile/ 等私有内容。
3
多终端访问 手机、平板、公司电脑通过浏览器随时访问，支持全文搜索和标签过滤。

图表绘制：Kroki.io + Diagram Code

遵循"代码即图表"原则，所有图表可编辑、可版本控制，禁止截图。

Mermaid

Obsidian 原生支持，用于流程图、时序图、状态图、甘特图。

PlantUML

UML 标准图表（类图、用例图、组件图），表达力强。

GraphViz

复杂依赖图、自动布局，适合包依赖、调用链。

D2

系统架构图、微服务拓扑、云基础设施。

# Kroki.io 渲染示例
curl -X POST https://kroki.io/mermaid/svg \
  -d 'graph TD; A-->B; B-->C;' \
  -o diagram.svg

效果：知识库在任何设备上可通过浏览器访问，图表支持实时预览和导出，便于分享和协作。

6. 富文本生成的务实方案

问题：没有订阅 image2、nano banana 等图片生成模型，如何制作精美的幻灯片、音视频？

解决方案：AI生成素材 + 手动导入专业工具

1
AI 生成 Markdown 素材 让 AI 生成结构化的 Markdown 内容（幻灯片大纲、演讲稿、知识点）
2
手动导入 Gemini Canvas 将 Markdown 导入 Gemini Canvas，利用其设计能力生成精美幻灯片
3
或使用 NotebookLM 将笔记导入 NotebookLM，生成音频播客、视频摘要

工作流对比

方案A：Gemini Canvas

适合：幻灯片、信息图、视觉化内容

流程：AI生成MD → 导入Canvas → 调整设计 → 导出PPT/PDF

方案B：NotebookLM

适合：音频播客、视频摘要、复习材料

流程：上传笔记 → 生成Audio Overview → 下载MP3/视频

方案C：本幻灯片

适合：技术分享、内部培训

流程：AI生成HTML → 浏览器打开 → 键盘翻页 → 投屏分享

核心理念：
不追求全自动，而是找到成本最低的协作点。AI负责内容生成，专业工具负责视觉呈现，人工负责质量把控。
