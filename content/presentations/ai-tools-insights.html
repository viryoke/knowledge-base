AI工具使用心得

构建个性化 AI 工作流的六个关键实践

viryoke 2026-06 Hermes Agent + Obsidian + Quartz
BACKGROUND

问题意识

AI 工具种类繁多，但工具太多、输出风格不一、知识无法积累。

核心思路：以知识库为中心，以全局配置为约束，让 AI 工具服务于长期知识积累。

痛点

  • 01 输出风格不一致
  • 02 AI 不懂个人偏好
  • 03 素材散落无法复利
  • 04 重复造轮子
  • 05 跨端共享不便

PRACTICE 02

让AI工具更懂你

通过个人经验积累、个人品味和持久记忆，让AI工具渐进式理解你的工作流。

个人经验积累

记录工作心得、最佳实践、踩坑经验

# 经验记录示例
scenario: 高并发场景
lesson: 缓存策略比查询优化更重要
context: 订单系统峰值处理
applied: 后续类似场景优先考虑

个人品味偏好

定义代码风格、架构偏好、设计原则

# 个人品味定义
code_style: 简洁优先，避免过度抽象
architecture: 偏好事件驱动而非轮询
documentation: 中文注释，英文代码
testing: 集成测试优先于单元测试

精准建议

每次经验记录都是知识资产。通过持久记忆机制，AI 能快速理解个人上下文，给出更符合个人品味和经验的精准建议。架构设计积累只是经验和品味积累的一部分。

PRACTICE 03

积累AI产生的素材

基于 Karpathy LLM Wiki 理念构建本地知识库，让素材形成复利网络。参考 Google 最新提出的 OKF（Open Knowledge Format） 标准。

LLM Wiki 核心理念

传统 RAG 每次从零发现知识，LLM Wiki 则一次编译、持续更新。交叉引用已建好，矛盾已标记，综合反映所有已摄入内容。知识库价值 = 页面数 × 交叉链接密度 × 内容质量。

Kroki.io 图表渲染引擎

统一的 Diagram as Code 渲染服务，支持 20+ 种图表语言。

# 渲染原理
1. 编写图表代码（Mermaid/PlantUML/D2）
2. Base64 编码（deflate + base64）
3. HTTP 请求 Kroki API
4. 返回 SVG/PNG 渲染结果

支持的图表语言：

Mermaid PlantUML D2 GraphViz Excalidraw C4 +15种

SUMMARY

核心理念：复利思维

实践 复利方式
统一配置 一次配置，所有工具受益
个人画像 一次描述，所有对话受益
知识库 一次摄入，所有查询受益
Skill 一次提炼，所有任务受益
跨端共享 一次发布，所有设备受益
富文本 一次编写，多种格式受益

AI 工具的最大价值不是"帮你写代码"，
而是帮你积累知识。
代码会过时，但知识会复利。
