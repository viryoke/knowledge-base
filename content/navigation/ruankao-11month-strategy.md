# 系统架构设计师 11月备考策略

## 诊断结论

基于5月考试复盘：
- **综合知识**：全面薄弱，有经验但大部分遗忘 → 需要系统性唤醒，非从零学习
- **案例分析**：答3题，质量属性/分层架构/端云协同有思路 → 基础框架在，需要补全盲区和练习答题规范
- **论文**：选了高并发性能提升，贴华为云经验但发挥不佳 → 需要模板化训练+提前准备3篇范文

## 核心原则

"唤醒"而非"重学"——你有12年工程经验和华为系统架构师背景，知识都在，只是生疏了。策略是快速激活，不是从头看教材。

## 两阶段计划

### 阶段一：续签前（6月1日 — 8月4日）· 轻量维持期

**每日投入：30-45分钟（利用班车时间）**

目标：综合知识唤醒 + 论文素材积累

**周计划循环（8周）：**

| 周次 | 综合知识主题 | 班车任务 | 周末任务 |
|------|------------|---------|---------|
| W1 | 计算机组成与体系结构 | 每天10道选择题+错题回顾 | 整理笔记到Wiki |
| W2 | 操作系统与并发 | 同上 | 同上 |
| W3 | 数据库系统 | 同上 | 同上 |
| W4 | 计算机网络 | 同上 | 同上 |
| W5 | 软件工程（需求/设计/测试） | 同上 | 开始写论文大纲1 |
| W6 | 系统架构风格与模式 | 同上 | 论文大纲2 |
| W7 | 信息安全 | 同上 | 论文大纲3 |
| W8 | UML建模+项目管理 | 同上 | 复习巩固 |

**论文准备（不写全文，只做大纲）：**
- 论文1：基于华为云云原生微服务架构设计（你最有把握的项目）
- 论文2：基于鸿蒙应用性能优化的架构调优（当前工作，素材最新）
- 论文3：高并发场景下的系统性能提升（5月已写过，优化版）

每篇大纲包含：
- 项目背景（300字）
- 遇到的问题/需求（200字）
- 架构设计决策与具体做法（800字，核心）
- 效果与反思（200字）

### 阶段二：续签后（8月5日 — 11月初）· 全力冲刺期

**每日投入：1.5-2小时（班车65分钟 + 晚间45分钟）**

目标：案例专项突破 + 论文全文写作 + 模拟实战

**月计划：**

**8月（案例分析突破）：**
- 每天1道案例真题精做（限时25分钟）
- 重点补盲区：数据流图、状态机、设计模式识别
- 建立案例答题模板（问题识别→原因分析→方案建议）

**9月（论文冲刺）：**
- 每周写1篇完整论文（限时120分钟）
- 3篇范文轮流打磨，确保考场可快速改写
- 请人review或让我帮你批改

**10月（模拟实战）：**
- 每周1套完整模拟卷（综合+案例+论文连考）
- 错题归类分析，针对性补强
- 论文背诵核心段落

## 我能帮你做的

1. **每日推送**：已设置。可以调整为按上述周计划推送对应主题的选择题
2. **知识点讲解**：遇到不理解的概念随时问我，我用你的华为项目经验做类比讲解
3. **案例批改**：你做完案例题贴给我，我按评分标准帮你批改
4. **论文批改**：你写完论文贴给我，我从结构/论点/字数/专业度四个维度评分
5. **Wiki沉淀**：重要知识点整理进Obsidian，形成你的个人知识网络

## 关键提醒

- 不要买太多教材——你有经验基础，需要的是"做题+查漏+唤醒"，不是"从头学"
- 论文是你的拿分关键——你的华为项目经验比90%的考生都强，只需要练好表达
- 综合知识75题只需45分（60%）——不需要全对，抓大放小
